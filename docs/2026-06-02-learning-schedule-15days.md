# 2026-06-02 — 정규과정 날짜 기반 메뉴 + 점검일(총 15일 일정)

## 배경
정규과정(`/learning/regular`)이 날짜 개념 없이 나열만 돼 있었음.
운영 일정(주말·공강 제외)에 맞춰 **날짜 기반 사이드바 메뉴 + 점검일 2일**을 반영하고,
강의 전체를 **Day 1~15(6/1~6/22)** 순차 번호로 확정.

## 1) 정규과정 날짜 기반 메뉴
- **`src/config/regularSchedule.ts`** — `REGULAR_DATES`(Day↔수업일), `fmtKDate`, `todayISO`.
- **`src/pages/Learning.tsx`** (정규과정 전용):
  - 사이드바 각 Day 옆 **수업 날짜 표기**(예: `Day 2 · HTML/CSS 기초  6/2(화)`).
  - **오늘 날짜의 Day 자동 선택·펼침** + 파란 **"오늘" 배지**, 접속일 기준 자동 진행.
  - 미래 Day **자유 접근**(잠금 없음).

## 2) 점검일 2일 — 사이드바 메뉴 항목으로 콘텐츠 포함
`regularTopics`에 점검일 토픽 2개 삽입(다른 Day와 동일하게 **Day 번호 부여**):
- **Day 4 · 1차 기초점검일**(6/5 금): HTML·CSS·JS 학습 점검 + 프로젝트 1차 팀별 회의.
- **Day 9 · 2차 학습점검일**(6/12 금): React·Supabase·AI·LLM API 연동 점검 + 프로젝트 2차 팀별 회의.
- 두 점검일 모두 사이드바 메뉴에서 클릭 시 점검 항목·회의 안건 콘텐츠 표시.
- 점검일 삽입에 따라 이후 콘텐츠 Day를 순차 재번호(React 기초 Day4→Day5 … 최종발표 Day13→Day15).
