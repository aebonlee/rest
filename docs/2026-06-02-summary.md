## 1. 🔴 관리자 명단/성적 0건 장애 → user_profiles RLS 무한재귀 수정 (`ad84604`)
- **증상**: 관리자 로그인은 되나 명단·성적이 텅 빔, 본인 이름 "이?" 반쪽 표시. 재로그인·세션초기화로도 복구 안 됨.
- **원인**: `user_profiles`에 남아있던 `user_profiles_select_own` 정책이 **자기참조 서브쿼리** 보유 → PostgreSQL **무한재귀(42P17)** → SELECT 전체 실패.
- **데이터 무손실 확인**: 가입 28명·선수평가 26건 보존(삭제 아님, 조회만 차단됐던 것).
- **해결(데이터 기반 관리자 모델)**:
  - `platform_admins`(전체 운영자) / `site_admins`(사이트 한정 관리자) 테이블 신설.
  - `is_platform_admin()`·`is_site_admin()` **SECURITY DEFINER** 함수(재귀 없음).
  - 단일 정책 `user_profiles_select_final`: 본인 + 운영자 + 사이트관리자.
- 배경: ax-study 등 디자인 카피 사이트가 같은 Supabase의 공유 `user_profiles`를 쓰며 PII 노출 우려 → 잠금 과정에서 옛 재귀 정책이 표면화된 것.
