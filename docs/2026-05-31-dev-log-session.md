## 1. 학습평가 개편 + 성적 Supabase 연동 (`53e3d08`)
- **선수평가**: 50→**20문항** 채점형, 문항당 5점(100점), 합격 **40점**
- **사후평가**: 50→**20문항** 최종 채점형, 합격 **60점**
- **진단평가**: 50문항 **자습 모드**(정답·해설 상시 공개, 채점 없음)
- 기존 50문항을 pool로 보존하고 `pick()`으로 대표 20문항 선별 (검증된 문항 재사용)
- 제출 시 `rest_assessments` 업서트 저장 → **대시보드(본인)** + **`/admin/grades`(관리자 전체 통계)** 연동
- 메뉴 명칭 통일: 선수평가 / 진단평가 / 사후평가

## 3. 다크모드 전면 정상화 (`190e2d6` `90eeffc` `27051c5`)
- **근본 원인**: 코드가 미정의 CSS 변수(`--bg-card`/`--bg-secondary`/`--border-color` 등) 사용 → 항상 밝은색 폴백 → 다크모드 깨짐
- **해결**: `base.css :root`에 미정의 변수들을 실제 테마 변수의 **별칭**으로 정의(`var()` 치환이 사용 지점 테마에서 해석되어 자동 대응) → 전 페이지 일괄 수정
- 네비 로고·메뉴 호버: `dark-mode.css`에 누락됐던 `[data-theme=dark][data-color=blue]` 추가(짙은 네이비가 다크 네비에서 안 보이던 문제)
- 모달 2종 하드코딩 흰 배경 변수화

## Supabase 신규 테이블 (운영 전 SQL 적용 필요)
`supabase/rest_tables.sql`에 정의:

| 테이블 | 용도 | 핵심 |
|--------|------|------|
| `rest_assessments` | 선수/사후 평가 성적 | UNIQUE(student_id,type), 본인+관리자 조회 |
| `rest_team_posts` | 팀별 비공개 게시판 | 팀 members 포함자+관리자만(RLS) |
| `rest_project_topics` | 학생 제안 주제 | 로그인 조회, 본인 작성 |
| `rest_topic_votes` | 주제 투표 | UNIQUE(user_id) 1인1표 |
| `rest_pledges` | 수강 다짐 | UNIQUE(user_id), 본인+관리자 |

> 출결은 기존 `rest_attendance` 사용(추가 불필요). 전 테이블 운영 Supabase에 적용 완료.
