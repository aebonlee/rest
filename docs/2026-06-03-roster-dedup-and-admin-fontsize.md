# 명단 동일인 통합 + 대시보드 폰트 축소 (2026-06-03)

## 개요
관리자 수강생 명단 대조(`/admin/roster`)에서 **주윤미가 명단외 가입에 누락**되던 문제를 해결하고,
**대시보드 폰트를 전체적으로 1~2px 축소**했다. 윤혜수 성별도 보정.

---

## 1) 주윤미 동일인 통합 + 명단외 누락 해결

### 증상
- "명단외 가입(2)"에 백진주·조두수(둘 다 rest 관리자)는 나오는데 **주윤미만 빠짐**.
- 주윤미는 이메일 2개(`jooym6016@kidico.or.kr`, `tlskaksmf@naver.com`)로 각각 가입.

### 원인 (점검으로 확정)
- 두 계정 모두 **`www.dreamitbiz.com`(본사이트)으로 가입** → AdminRoster의 도메인 조건
  (`signup_domain=rest` OR `visited_sites⊇rest`)에 안 걸려 **아예 로드되지 않음**.
- 또한 두 계정은 전화/이름이 달라(`tlskaksmf`는 Kakao·이름 없음) 기존 `groupByPerson`(전화→이름)으로 **동일인 통합 안 됨**.
- RLS상 본사이트 가입자는 전체 운영자(role admin/superadmin 또는 글로벌 이메일)만 조회 가능
  → aebon 계정으로 로그인하면 도메인 무관 조회 가능(확인됨).

### 해결
- `config/admin.ts` — **명시적 동일인 묶음** `SAME_PERSON_EMAIL_GROUPS` 추가
  (`{ name: '주윤미', emails: [jooym6016…, tlskaksmf…] }`).
- `utils/people.ts` — `personKey`가 동일인 묶음 이메일을 **최우선 키**로 사용(전화·이름보다 우선),
  `groupByPerson`이 묶음 지정 이름('주윤미')을 표시 이름으로 적용.
- `pages/admin/AdminRoster.tsx`:
  - 묶음 **이메일로 도메인 무관 추가 조회**(`.in('email', …)`) + **이름으로도 추가 조회**(`.or(name.eq…)`).
  - STAFF 역할이어도 묶음 이메일/이름은 **필터 예외**로 항상 포함.
  - (점검용) "동일인 진단" 패널을 임시 추가 → 검증 완료 후 제거.

### 결과
- 주윤미 2계정이 **1명 "주윤미"(동일인 2)**로 명단외 가입에 표시.
- "명단외 가입"·"가입 회원" 카운트가 **계정 수가 아닌 사람 수**로 일관 집계.
- 로그인(aebon) 상태에서 진단 박스에 2계정 정상 표기 확인 → 패널 제거.

### 작업 (커밋 `767e11e`)
- `styles/admin.css` — 사이드바 헤더 17→15 / 메뉴 16→14 / 아이콘 18→16,
  제목 24→22, 스탯카드 28→26·라벨 15→14, 테이블 16→14, 배지·출결상태·버튼 14→13, 입력 16→14.
- 인라인 `fontSize` (AdminAttendance·Grades·Roster·Students): 22→20, 16→14.5, 15.5→14,
  15→13.5, 14.5/14→13, 13.5→12.5, 13→12, 12/11.5→11.
- 누적 치환(16→14→13) 방지를 위해 **작은 값부터 순서대로** 치환.
- 학습(본문) 폰트는 유지, **대시보드(admin)만** 조정.
