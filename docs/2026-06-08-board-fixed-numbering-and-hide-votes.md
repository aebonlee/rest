# 2026-06-08 — 투표 보드 고정 번호(1~21)·패들렛 일치 + 득표 표시 숨김

## 배경
프로젝트 주제 보드(ProjectVote)에 프리셋 7 + 학생 제안 14 = **21개** 주제가 쌓이면서,
기존 번호(득표순 idx+1 → 이후 TEAM_PROJECTS.id)가 띄엄띄엄/뒤죽박죽으로 보임.
운영자 확정: **1~21 연속 고정 번호**로 매기고 패들렛 번호와 1:1 일치, 보드는 번호순 정렬.
이후 보드를 사실상 "번호순 팀 목록"으로 쓰기로 하여 **득표 수 표시는 숨김**(데이터·기능은 유지).

## 번호 순서 확정(1~21)
- 1~7: 프리셋(운영자 기본 주제)
- 8~21: 학생 제안 주제 **생성순(created_at)**
- 근거: "고정 번호"는 가변적인 득표순보다 안정적 기준(생성순)이 적합.

## 변경

- `BOARD_ORDER`(21개 제목, 인덱스+1 = 번호 = 패들렛 project 번호) 단일 진실 공급원.
- `getBoardNo(title)` / `BOARD_SIZE` export. 제목은 정규화(공백·대소문자 무시) 비교.

### `src/pages/ProjectVote.tsx`
- 카드 번호·패들렛 링크·정렬을 **득표순 → 고정 보드 번호**(getBoardNo)로 전환.
  - 미등록 신규 주제는 22번부터 생성순 임시 부여.
  - `rows` 정렬을 득표순 → 번호순(1→21→신규)으로 변경.
- **득표 표시 숨김**: 상단 "총 N표", 카드 우측 큰 득표수, 막대그래프, 투표자 칩 제거.
